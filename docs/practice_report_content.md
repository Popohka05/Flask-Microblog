# Отчет по практике: Flask Microblog

## 1. Выбранный проект

Название проекта: Flask Microblog

Ссылка на проект из каталога: https://github.com/practical-tutorials/project-based-learning

Исходный учебный материал: https://blog.miguelgrinberg.com/post/the-flask-mega-tutorial-part-i-hello-world

Краткое описание:
Flask Microblog представляет собой веб-приложение для публикации коротких сообщений и взаимодействия между пользователями. Пользователь может зарегистрироваться, войти в систему, оформить профиль, создавать записи и просматривать ленту публикаций. Приложение ориентировано на демонстрацию базовых паттернов full-stack веб-разработки на Python с использованием серверного рендеринга. В ходе практики проект был адаптирован под требования учебной отчетности и публичного размещения в GitHub.

Что реализовано / доработано в ходе практики:

- создано веб-приложение на Flask с модульной структурой проекта;
- реализована регистрация пользователей;
- реализована авторизация и выход из системы;
- добавлены профили пользователей с возможностью редактирования;
- реализовано создание публикаций;
- реализовано редактирование и удаление собственных публикаций;
- реализована подписка на других пользователей и отписка;
- добавлена персонализированная лента на основе подписок;
- реализован поиск по пользователям и тексту публикаций;
- добавлены тесты для ключевых пользовательских сценариев;
- подготовлены демо-данные для проверки проекта;
- оформлен README и конфигурация для деплоя.

## 3. Архитектура проекта

### 3.1 Общая архитектурная схема

Пользователь взаимодействует с HTML-интерфейсом, который формируется сервером Flask через шаблоны Jinja2. Серверная часть обрабатывает маршруты, формы, бизнес-логику публикаций, подписок и авторизации. Данные пользователей, записей и связей подписки сохраняются в SQLite через Flask-SQLAlchemy.

Схема взаимодействия:

`Пользователь -> Flask routes -> SQLAlchemy models -> SQLite database`

### 3.2 ERD

Сущности и связи:

- `User`
  - `id`
  - `username`
  - `email`
  - `password_hash`
  - `about_me`
  - `created_at`
- `Post`
  - `id`
  - `body`
  - `created_at`
  - `author_id`
- `followers`
  - `follower_id`
  - `followed_id`

Связи:

- один пользователь может иметь много публикаций;
- одна публикация принадлежит одному пользователю;
- между пользователями реализована связь многие-ко-многим через таблицу `followers`.

### 3.3 Use Case

Основные пользовательские сценарии:

1. Регистрация и вход в систему.
2. Создание, редактирование и удаление публикации.
3. Просмотр профиля пользователя и его публикаций.
4. Подписка на другого пользователя и просмотр персональной ленты.
5. Поиск пользователей и публикаций по ключевому слову.

### 3.4 API / маршруты приложения

Так как приложение построено на серверном рендеринге, ключевые маршруты представлены в виде web-endpoints:

- `/` — главная страница и лента публикаций;
- `/register` — регистрация пользователя;
- `/login` — авторизация;
- `/logout` — выход из системы;
- `/users` — список пользователей;
- `/user/<username>` — профиль пользователя;
- `/profile/edit` — редактирование собственного профиля;
- `/posts/create` — создание публикации;
- `/posts/<id>/edit` — редактирование публикации;
- `/posts/<id>/delete` — удаление публикации;
- `/follow/<username>` — подписка на пользователя;
- `/unfollow/<username>` — отписка от пользователя;
- `/search` — поиск по пользователям и публикациям.

## 7. Вывод по практике

В ходе производственной практики был разработан и оформлен веб-проект Flask Microblog на языке Python с использованием микрофреймворка Flask. В процессе работы была изучена организация серверного веб-приложения, включающего маршрутизацию, обработку форм, авторизацию пользователей, работу с шаблонами и взаимодействие с базой данных через ORM. В результате был создан полнофункциональный учебный сервис микроблогов, поддерживающий регистрацию пользователей, создание публикаций, редактирование профиля, поиск и механизм подписок.

Наиболее значимой частью практической работы стала реализация логики взаимодействия между пользователями. Особое внимание уделялось хранению и обработке данных, настройке моделей `User` и `Post`, а также организации связи многие-ко-многим для подписок. Дополнительно были реализованы тесты для проверки критически важных пользовательских сценариев: регистрации, авторизации, публикации записи и отображения персональной ленты.

Во время выполнения проекта были освоены навыки структурирования Flask-приложения, базового тестирования, подготовки README-документации и адаптации проекта под требования учебной практики. Также была проведена подготовка проекта к деплою и дальнейшей публичной демонстрации. В перспективе проект можно улучшить путем перехода на PostgreSQL, добавления пагинации, расширения профилей пользователей и внедрения более подробной системы модерации контента.
